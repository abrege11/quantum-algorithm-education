Interestingly enough, there is no Bernstein-Vazirani algorithm. The Deutsch-Jozsa algorithm solves the Bernstein-Vazirani exactly how it is, the only differnce being when you stop the algorithm.

Note that a recursive approach was created, however, we will simply see what the standard Deutsch-Jozsa approach looks like.

We will run through this algorithm step by step as usual, however, if you want to see what the Bernstein-Vazirani algorithm does differently, you can click the link above to skip to that section.

The Bernstein-Vazirani problem is simple, there is a hidden string of binary digits that is size \( n \), and you have to guess what that stream of digits is.

With this algorithm we still see a stream of binary digits as input, represented as: \[ f :\{0,1\}^n \longrightarrow \{0,1\} \] Classically, we would solve this by passing in strings of all 0's with a singular 1 until our string emerges, below is a graphic displaying this method


This solution is taking the bitwise inner product with modulo 2, don't act like you know what that means, let's break it down \[ f(x) = s \cdot x \mod 2 \] What does this mean? When \( n=4 \), a possible secret string \( s \) could be \( 1001 \), which means

\( s_1 = 1\)
\( s_2 = 0\)
\( s_3 = 0\)
\( s_4 = 1\)

So, let's build our \( x \) query string and take the bitwise inner product with our \( s \) string

As we did in the example, let's start with \( x = 0001 \), which means

\( x_1 = 0\)
\( x_2 = 0\)
\( x_3 = 0\)
\( x_4 = 1\)

Now, we multiply each value and add them together, \[ (x_1 \times s_1) + (x_2 \times s_2) + (x_3 \times s_3) + (x_4 \times s_4) \longrightarrow \] \[ (1 \times 0) + (0 \times 0) + (0 \times 0) + (1 \times 1) \longrightarrow \] \[ 0 + 0 + 0 + 1 \longrightarrow \] \[ 1 \] Now, we still have to fulfill the \( \mod 2\) portion of the function \[ \mod 2 (1) = 1 \] And just like that, we know the 0001 value of our string, which is 1.

Now let's find the 0010 value. \[ (x_1 \times s_1) + (x_2 \times s_2) + (x_3 \times s_3) + (x_4 \times s_4) \longrightarrow \] \[ (1 \times 0) + (0 \times 0) + (0 \times 1) + (1 \times 0) \longrightarrow \] \[ 0 + 0 + 0 + 0 \longrightarrow \] \[ 0 \] and of course, \(\mod 2 (0) = 0\), so the 0010 position of our string is 0. Now our string sits at \( 01 \).

Let's continue and find our last two spots. \[ (x_1 \times s_1) + (x_2 \times s_2) + (x_3 \times s_3) + (x_4 \times s_4) \longrightarrow \] \[ (1 \times 0) + (0 \times 1) + (0 \times 0) + (1 \times 0) \longrightarrow \] \[ 0 + 0 + 0 + 0 \longrightarrow \] \[ \mod 2 (0) = 0 \] which gives us 001 and \[ (x_1 \times s_1) + (x_2 \times s_2) + (x_3 \times s_3) + (x_4 \times s_4) \longrightarrow \] \[ (1 \times 1) + (0 \times 0) + (0 \times 1) + (1 \times 0) \longrightarrow \] \[ 1 + 0 + 0 + 0 \longrightarrow \] \[ \mod 2(1) = 1 \] Which finally leaves our output string at 1001.

This method works, but it's slow, and we now know that quantum algorithms are superior, so we can leave this algorithm in the dust, and complete the problem in just a singular query.

This is where the Bernstein-Vazirani problem differs from the Deutsch-Jozsa problem. See, we don't need to worry about the post processing of our data, we actually are targeting the string of bits.
